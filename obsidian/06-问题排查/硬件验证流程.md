# 硬件验证流程

## 总原则

AD9144 是 JESD204 高速链路，先用 ILA 证明数字链路接通，再用示波器看模拟输出。当前 TX 数字侧已经通过基础检查，可以进入示波器首测。

## 前置条件

- FMCADDA-9250-9144 子卡插稳并上电。
- K325T 正常上电。
- Vivado 2024.1 能连接 `xc7k325t_0`。
- 已生成并烧录 `D:\FPGA\ad9144_bringup_k325t\vivado\top_direct.bit` 或 cfg debug bit。
- 烧录后等待 12-15s，让 vendor reset/config 序列完成。

## 烧录后检查

1. Hardware Manager -> Open Target -> Auto Connect。
2. Program Device，选择最新 bit 文件。
3. 确认 DONE 指示。
4. 等待 12-15s。
5. 确认 debug cores 存在并可 arm。

## TX ILA 基线

| 检查项 | 当前观测 | 含义 |
|---|---|---|
| QPLL lock | 1 | GTX refclk/PLL 正常 |
| TX reset done | 1 | GTX/JESD TX reset 释放完成 |
| TX tready | 1 | JESD TX 接收 sample data |
| SYNC status | 1/稳定 | AD9144 同步状态正常 |
| SYSREF | 0/1 observed | Subclass 1 参考有活动 |
| TX sample data | 变化 | ROM 数据进入链路 |

## 示波器检查

当前第一版预期不是最终 AWG DDS，而是 vendor ROM waveform。

- 先测 OUT1。
- 使用 50 ohm 负载/终端。
- 预期频率约 50MHz。
- OUT1 为平线时，依次试 OUT2-OUT4。

如果所有输出都是平的：

1. 复查 TX ILA 是否仍满足基线。
2. 看 SYSREF 和 SYNC 是否仍有活动/稳定。
3. 看 AD9144 SPI 初始化是否完成。
4. 查模拟端输出使能、供电、时钟/PLL 指示灯、连接线和示波器量程。
